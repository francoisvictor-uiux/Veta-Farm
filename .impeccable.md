# Veta Farm — Design Context

## Design Context

### Users
Multi-role internal enterprise tool used by: Farm Manager, Sales Officer, Purchasing Officer, Accountant, Veterinarian, Warehouse Manager. Users work primarily on desktop in an office/farm-office context. The interface is in Arabic (RTL) and must accommodate users with varying technical proficiency. Sessions are task-focused and repetitive — users return daily to complete operational workflows.

### Brand Personality
**Three words**: Modern · Clean · Professional

The brand is rooted in agriculture (forest green #1a6b3c) with a Mindaro lime accent. It should feel like well-crafted enterprise software — not a startup product, not a government portal. Confidence-inspiring, efficient, and calm.

### Aesthetic Direction
- Enterprise SaaS quality: think well-crafted ERP/B2B tools
- Calm and functional — no decorative flourishes, no personality gimmicks
- RTL-first layout (Arabic) with Cairo typeface
- Light mode primary, dark mode optional
- Clean data-dense layouts — users need to see a lot of information at once
- Anti-references: avoid consumer-app playfulness, avoid heavy glassmorphism, avoid gradient-heavy "AI aesthetic"

### Design Principles
1. **Clarity over decoration** — Every element earns its place. If it doesn't help the user complete a task, remove it.
2. **Data density with breathing room** — Users work with tables and forms all day. Pack information efficiently but don't crowd it.
3. **Consistent affordances** — Every interactive element should look and behave predictably across all roles.
4. **Accessible by default** — WCAG AAA target. RTL semantics, proper ARIA, keyboard navigation, sufficient contrast.
5. **Professional restraint** — The aesthetic should signal expertise and reliability, not trend-following.
